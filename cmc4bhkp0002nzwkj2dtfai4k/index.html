Loop ワークスペースの「メンバーの招待と管理」でメンバーの役割を変更するとワークスペースにアクセス出来なくなる場合がある

Last Update: 2025-06-20

こんにちは、Office サポートチームです。
この記事では、Loop ワークスペースの「メンバーの招待と管理」でメンバーの役割を変更するとワークスペースにアクセス出来なくなる場合がある事象について説明します。


事象

Loop のメインページからワークスペースを開いた場合に以下のようなメッセージが表示されます。

Loop ワークスペースのリンクを直接開いた場合、以下のようなメッセージが表示されます。

「メンバーの招待と管理」で削除し再追加しようとした場合、「新しいメンバーを正常に追加しました」と表示されても、メンバーに追加されていません。
※ 見た目上、追加されるケースも確認されておりますので、この事象は生じない場合もございます。


発生条件

以下の条件を満たした場合に生じることが確認されております。

発生条件 1) Loop ワークスペースの「メンバーの招待と管理」でメンバーの役割を変更する。

発生条件 2) ※調査中※

本条件は現在も調査中でございますが、2025/5/30 以降に作成した新規ワークスペースでは事象の発生を確認しておりませんため、
2025/5/30 より前に作成した Loop ワークスペースにて事象が生じる可能性がございます。

Loop ワークスペースの作成日は、ユーザーは UI から確認することが出来ませんが、
ワークスペース内の各ページのバージョン履歴を確認することで、目安にしていただけます。

しかしながら、ページが削除されている可能性がございます場合、この方法はご利用いただけませんため、
例えば、テナントの全体管理者、または、SharePoint Embedded管理者にて、
SharePoint 管理センターより、Containers – Active containers から該当のコンテナーを開き、全般 - 作成日を確認する必要がございます。

しかしながら、現在も調査中の状況のため、影響が大きいワークスペースについては、
新規ワークスペースについても役割変更自体をお控えいただけますと幸いです。


回避策について

ワークスペースの単位の事象につきましては、問題が生じたワークスペースに関しては再作成する以外の回避策をご提供出来ない可能性がございますため、
恐れ入りますが、可能でございましたらワークスペースの再作成をご検討いただけますと幸いです。

なお、ワークスペースの再作成以外の回避策といたしましては、右上の “共有” から、ページ単位での共有を行っていただくことが挙げられます。


今後について

現在、弊社開発部門で調査を行っておりますが、具体的なスケジュールは未定となります。
誠に恐れ入りますが、上記回避策にてご対応をいただくか、弊社開発部門での調査をお待ちくださいますようお願いいたします。


本情報の内容 (添付文書、リンク先などを含む) は、作成日時点でのものであり、予告なく変更される場合があります。
